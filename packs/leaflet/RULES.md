# Leaflet

Portable, project-agnostic gotchas for the [Leaflet](https://leafletjs.com/) web-mapping library — loading it from a CDN, embedding a map in a page, and the marker/`divIcon` and plugin behaviours that bite regardless of which app renders the map. True for any Leaflet map read cold.

- **Load Leaflet — and every Leaflet plugin — from a version-pinned CDN URL with a Subresource-Integrity hash and `crossorigin`.** Pin the exact version (`leaflet@1.9.4`, not `@latest`) and carry `integrity="sha256-…"` + `crossorigin=""` on both the CSS `<link>` and the JS `<script>`, so an upgraded or tampered CDN asset can't silently change under the app. The trap is doing this for the core bundle but forgetting a plugin: pin and hash the plugin the same way, or the one un-pinned `<script>` is the hole. (Grounded in `index.html`, which pins+SRIs `leaflet@1.9.4` core yet loaded `leaflet.markercluster@1.5.3` without an `integrity` attribute — the inconsistency is the lesson.)

- **Feature-detect an optional Leaflet plugin before using it, and fall back to core when its CDN script didn't load.** A separate `<script>` for a plugin (markercluster, heat, draw, …) can silently fail to load while the base Leaflet bundle succeeds, leaving `L.markerClusterGroup` (etc.) undefined. Guard the call with `typeof L.markerClusterGroup === "function"` and, when it's absent, add markers straight to the map instead of throwing — the base map must still render. (Grounded in `js/app.js` `renderMarkers()`: the cluster group is created only behind the `typeof` guard, and each marker falls back to `marker.addTo(map)` when the plugin is unavailable.)

- **Default an embedded, mid-page map to `scrollWheelZoom: false` so it doesn't hijack the page scroll.** A map that isn't the whole viewport but grabs the wheel traps a reader who scrolls onto it. Construct it with `L.map(el, { scrollWheelZoom: false })` (or gate wheel-zoom behind a click/focus). (Grounded in `js/app.js` `initMap()`: `L.map("leaflet-map", { scrollWheelZoom: false })`.)

- **Keep the tile provider's attribution — it's a licence term, not decoration.** OpenStreetMap's tile-usage policy requires visible attribution, so the `attribution: '… © OpenStreetMap contributors'` on the `L.tileLayer(...)` must stay even when tidying the UI; stripping it breaks the licence. Set `maxZoom` to the provider's real ceiling (OSM tiles top out at 19) so Leaflet doesn't request zoom levels the provider doesn't serve. (Grounded in `js/app.js` `initMap()`: the OSM `tileLayer` carries the attribution string and `maxZoom: 19`.)

- **When a marker needs its own CSS `transform`, put it on an inner element, not the `divIcon` root — Leaflet writes an inline `transform` on the icon root to position it.** Leaflet positions every marker by writing an inline `transform: translate3d(...)` onto the icon's root element; a `transform` you set on that same root (to rotate or centre the glyph) is clobbered on the next reposition. Make the icon root a zero-size anchor (`iconSize: [0, 0]`, `iconAnchor: [0, 0]`) and apply your rotation/centring transform to a `<span>` inside its `html`, whose transform Leaflet never touches. (Grounded in `js/app.js`: the route arrowhead rotates an inner `.route-arrow` span, and the arrival-check pin uses a zero-size anchor root with the visible pill as an inner span, exactly to survive Leaflet's root transform.)
